# 引き継ぎ書【Opus用】バフェットかおる「答え合わせ切り抜き動画」制作

あなた（Claude Opus）はこの文書を読んで、前任のClaude（Fable）が行っていた
YouTube切り抜き動画の制作を引き継ぎます。あなたには判断力があるので、この文書は
「手順の暗記」ではなく「**何のために・どういう基準で判断するか**」を中心に書いています。
迷ったら §2 の品質基準に立ち返って自分で判断してください。ただし §7 の
「絶対ルール」だけは判断の余地なく守ってください。

同フォルダの `HIKITSUGI.md`（初代の引き継ぎ書）と `HIKITSUGI_SONNET.md`（手順重視版）も
参照できます。コマンドの完全なコピペが欲しいときはSONNET版が便利です。

## 1. 仕事の目的と全体像

依頼主は投資系YouTuber「バフェットかおる」さん（登録者27,500人・毎晩21時ライブ配信・
元JAL客室乗務員・資産1.5億円超・年間配当500万円超）。収益は月7万円程度で、
視聴回数を伸ばすことが目標。

あなたの仕事は、かおるさんの過去のライブ配信から**視聴者が本当に見たい部分**
（銘柄の説明・実際の売買・配分の公開・予想の的中）を切り抜き、
**「答え合わせアイキャッチ」**を焼き込んで、視聴者が2秒で「この人当たってる」と
わかる動画に仕上げることです。

### なぜ「答え合わせ」なのか
かおるさんの核メッセージ：
> 高配当株はAIや半導体のように乱高下しない代わりに、上下を繰り返しながら着実に右肩上がり。
> 「増配」で配当は追加投資なしでも育ち、いずれ配当だけで元本を回収できる。
> 老後は"タダで増える配当"を受け取るだけになる。

これを最も説得力ある形で示せるのが「過去の映像 × 現在の数字」です。
過去のライブ映像には当時の株価・日付・実際の買い注文が映っており、それ自体が
**改ざん不可能な証拠**になります。数字は実績なので盛る必要がなく、
負けた銘柄も隠さず載せることで信頼が生まれます（例：4銘柄中「竹本容器+7%」だけ
灰色で正直に表示した → 視聴者の信頼向上）。

### 発信スタンス（かおるさん本人の意向）
- 自慢ではなく事実の提示。「私の考えがすべて正しいわけではない、参考にしてください」
  という姿勢を概要欄に必ず明記する
- 計算は概算であることを明記。追加投資（割安時に数株ずつ買い増し）をしているため
  実際の損益とは差がある旨も書く
- 出典を明記する（動画内で視聴している番組：松井証券公式YouTube
  https://www.youtube.com/channel/UCS5dVIXy8SYcIMWRmEI-_3A、
  ダイヤモンドZAiなどの経済メディア、ネット上の公開情報）

## 2. 品質基準（迷ったらここに立ち返る）

1. **数字の正確さ > すべて**。1つの計算ミスが全信頼を壊す。必ず電卓（python3）で検算し、
   検算結果の表をユーザーに見せる。暗算は禁止（前任は暗算で合計を間違え指摘された）
2. **日付の明確さ**。視聴者が「これはいつの映像か」「いつの株価での答え合わせか」を
   混同しない構成にする（§7参照）
3. **視聴者ファースト**。アイキャッチが本編の字幕・重要情報を隠していないか、
   実際のフレームを自分の目で確認する
4. **誠実さ**。概算には「約」を付ける。不明な数字は作らない。負けも載せる
5. **ユーザーの手間の最小化**。かおるさんはITが得意ではない。案内は番号付き・
   リンク付き・クリックする場所まで具体的に。詰まったら「画面のスクショを送ってください」

## 3. 環境の制約（技術的な前提）

- クラウドのClaude Code環境は**YouTube・Googleドライブ・大半の外部サイト接続不可**
  （プロキシが403）。使えるのは github.com / objects.githubusercontent.com / WebSearch のみ
- WebSearchの株価は日付が古い・混在があり信用しない。**ユーザーのSBIスクショを正**とする
  （前任はヒューリックで検索値1,875円を使いかけたが、実際は1,734.5円だった）
- ffmpeg / fonts-noto-cjk は `apt-get install -y ffmpeg fonts-noto-cjk` で導入
- ユーザーのMacにはローカルのClaude Codeがあり、**YouTubeからのダウンロードは
  そちらに依頼できる**（yt-dlpコマンドを渡す）。Studio のダウンロードは
  著作権申し立て付き動画・回数制限（1日5回）で失敗することがある

### 動画の受け取り（ユーザー → あなた）
ユーザーが https://github.com/buffettkaoru/ending-note/releases/new にタグを作って
ドラッグ&ドロップ →「Publish release」（**Publishまで押さないと取得できない**。
下書き止まりのことが過去2回あった。その場合の案内文はSONNET版§2に定型あり）。
取得: `curl -sSL --cacert /root/.ccr/ca-bundle.crt -o video.mp4 "https://github.com/buffettkaoru/ending-note/releases/download/<タグ>/<ファイル名>"`
日本語タグはURLエンコードする（例:「動画」→ %E5%8B%95%E7%94%BB）。
一覧確認: `.../releases/expanded_assets/<タグ>` をgrep。

### 納品（あなた → ユーザー）
SendUserFileのカードはユーザー環境で開けないことがある。**GitHub rawリンク方式が確実**：
1. 完成mp4を `deliverables/` に**ASCIIファイル名・100MB未満**でコミット＆プッシュ
2. `https://github.com/buffettkaoru/ending-note/raw/<ブランチ>/deliverables/<名前>` を案内
   （プッシュ直後は404のことがある。15秒待って再確認）
3. ユーザーの「できた」を確認したら `git rm` でお掃除（**約束したお掃除は必ず実行**）

### 監視
send_later（45〜60分間隔）でreleasesを定期チェック。変化がなければユーザーに
通知せず静かに再スケジュール。トリガーのメッセージには「現在の状況・次にやること・
学んだ制約」を毎回書き込み、未来の自分への引き継ぎにする。

## 4. 制作ワークフロー（判断ポイントつき）

1. **受領**: ダウンロード → ffprobe（長さ・解像度・音声を確認）
2. **フレームスキャン**: 動画全体を等間隔（60分超なら3〜4分、10分以下なら30秒）で
   フレーム抽出し、タイムスタンプを焼き込んで4×5タイルにまとめ、**Readで自分の目で見る**。
   探すもの：(a)構成・話題の切り替わり (b)銘柄名・スライド (c)**当時の株価・日付が映った
   証拠画面**（SBIの画面には日時が出る。これが答え合わせの証拠力の源泉）
   (d)実際の売買・入金画面 (e)ポートフォリオ全体画面
3. **切り抜き区間の判断**（あなたの編集センスの見せ所）:
   - 最優先: 銘柄を一気に説明する区間（配分表・銘柄リスト）と実際の売買シーン
   - 視聴者アナリティクス（チャットの盛り上がり・視聴維持率）のスクショがもらえたら
     その山を優先
   - 通常動画は8分超（ミッドロール広告可）、ショートは60秒以内推奨・3分以内必須
   - 境界は必ず `silencedetect`（noise=-30dB:d=0.4）で文の切れ目にスナップ。
     スライドの切り替わりも±10秒のフレーム抽出で確認してから決める
4. **数字の収集と検算**: 当時値=映像内画面 ＞ ユーザーのスクショ ＞ WebSearch（要確認と明記）。
   現在値・配当・株数=ユーザーのスクショ。全計算をpython3で行い、表で提示
5. **カット**: 編集素材は `-c copy` の無劣化。**アイキャッチは投稿直前の完成版にだけ焼き込む**
   （§7の教訓1）
6. **アイキャッチ焼き込み**: §5のデザインシステムに従う。文言はコンテンツごとに
   あなたが考えてよいが、要素の役割（左上=正体・右上=数字・日付=必須）は変えない
7. **検証**: 冒頭・中間・末尾のフレームをReadで目視（誤字・重なり・数字・
   本編の字幕が隠れていないか）。ffprobeで長さ・サイズ確認
8. **納品**: rawリンク + タイトル・概要欄・サムネ文言 + （頼まれたら）ツイート文。
   概要欄はテンプレ（§6）をベースにその動画の数字で埋める
9. **お掃除**: DL確認後にdeliverables削除。リリースの素材も削除を勧める

## 5. アイキャッチのデザインシステム

### 設計思想
- **左上 = この動画の正体**（何の答え合わせか・映像はいつか・誰が語るか）
- **右上 = 数字の証拠**（銘柄ごとの当時→現在、騰落率、本人の実額）
- **下部 = 使わないのが原則**。本編の字幕・番組テロップと衝突するため。
  日付などの必須情報も左上に統合する（§7の教訓2）
- 色の意味: 赤=正体バッジ / 黄=見出し / 緑=勝った数字 / 金=配当・お金の総額 /
  灰=負け・横ばい（正直さの演出）/ 濃紺黒(0x101018)=背景パネル
- 過去に焼き込んだ古いアイキャッチを上書きする場合は、不透明(@0.97)の
  drawboxパネルを先に敷いてから新要素を描く

### 実寸（720p基準。他解像度は 高さ/1080 でスケール）
フォント: `FONT=$(fc-list :lang=ja --format '%{file}\n' | grep -i bold | head -1)`
（Noto Sans CJK Bold）。drawtextは**必ず textfile= + expansion=none**
（直書きは % & : が化ける）。テキストは `printf '文言' > txt/tN.txt` で用意。

- 左上バッジ: fontsize 31-38, x=27, y=24〜, 赤 0xCC0000@0.9〜0.95
- 左上サブ行(日付): fontsize 21, 白文字（背景パネルの上に直置き）
- 左上強調行(バフェット等): fontsize 23, 金文字 0xFFD24A
- 右上見出し: fontsize 28-30, yellow@0.92, 黒文字, 右寄せ x=w-text_w-24
- 右上銘柄行: fontsize 26-28, 0x0B6E2F@0.88, 43-48px間隔で縦積み
- 右上お金の行: fontsize 25-27, 0xFFC93C@0.9〜0.98, 黒文字
- boxborderw はフォントサイズの約1/3

### 実コマンド（前回の完成版そのまま。数字と文言を差し替えて使う）
```bash
D="fontfile='$FONT':expansion=none:fontcolor=white"
ffmpeg -y -i input.mp4 -vf "\
drawbox=x=10:y=10:w=560:h=200:color=0x101018@0.97:t=fill,\
drawbox=x=iw-480:y=10:w=470:h=380:color=0x101018@0.97:t=fill,\
drawtext=$D:textfile=txt/L1.txt:fontsize=31:box=1:boxcolor=0xCC0000@0.95:boxborderw=9:x=27:y=24,\
drawtext=$D:textfile=txt/L2.txt:fontsize=21:x=27:y=82,\
drawtext=$D:textfile=txt/L3.txt:fontsize=23:fontcolor=0xFFD24A:x=27:y=122,\
drawtext=$D:textfile=txt/L4.txt:fontsize=23:fontcolor=0xFFD24A:x=27:y=158,\
drawtext=$D:textfile=txt/R0.txt:fontsize=28:fontcolor=black:box=1:boxcolor=yellow@0.95:boxborderw=8:x=w-text_w-24:y=22,\
drawtext=$D:textfile=txt/R1.txt:fontsize=26:box=1:boxcolor=0x0B6E2F@0.9:boxborderw=7:x=w-text_w-24:y=70,\
drawtext=$D:textfile=txt/R2.txt:fontsize=26:box=1:boxcolor=0x0B6E2F@0.9:boxborderw=7:x=w-text_w-24:y=113,\
drawtext=$D:textfile=txt/R3.txt:fontsize=26:box=1:boxcolor=0x0B6E2F@0.9:boxborderw=7:x=w-text_w-24:y=156,\
drawtext=$D:textfile=txt/R4.txt:fontsize=26:box=1:boxcolor=0x0B6E2F@0.9:boxborderw=7:x=w-text_w-24:y=199,\
drawtext=$D:textfile=txt/R5.txt:fontsize=25:fontcolor=black:box=1:boxcolor=0xFFC93C@0.95:boxborderw=7:x=w-text_w-24:y=252,\
drawtext=$D:textfile=txt/R6.txt:fontsize=25:fontcolor=black:box=1:boxcolor=0xFFC93C@0.95:boxborderw=7:x=w-text_w-24:y=294,\
drawtext=$D:textfile=txt/R7.txt:fontsize=27:fontcolor=black:box=1:boxcolor=0xFFC93C@0.98:boxborderw=8:x=w-text_w-24:y=334" \
-c:v libx264 -preset veryfast -crf 21 -pix_fmt yuv420p -c:a copy -movflags +faststart out.mp4
```
文言の実例（銀行株回の完成版）: L1「1年半前の銀行株予想 答え合わせ」
L2「映像:2025年2月17日ライブ／作成:2026年7月5日」L3「この後バフェットが東京海上に2,874億円出資」
L4「→株価は急騰し上場来高値に」R0「銀行・保険株の答え合わせ」
R1「三菱UFJ 1,983→3,326円 +68%」…R5「私の保有分：値上がり +284万円」
R6「配当2年分 +30万円」R7「合計 +314万円に」

縦型ショート(1080×1920)は同フォルダ `extract_short.sh`（上下ぼかし・音量正規化つき）。

## 6. タイトル・概要欄・サムネ・ツイートの型

- **タイトル**: 【答え合わせ】＋実額 or 実%＋期間。
  例:「【答え合わせ】1年半前に注目した銀行株、保有4銘柄で＋314万円になっていました」
- **概要欄の必須要素**（順に）: ①いつの映像か＋動画作成日 ②銘柄ごとの当時→現在
  ③配当の実績（増配） ④本人の実額（概算と明記） ⑤バフェット等の後日談
  ⑥計算の注記（概算・現在の株数ベース・割安時に数株ずつ買い増ししている）
  ⑦出典（松井証券公式YouTubeのリンク・ZAi等） ⑧「私の考えがすべて正しいわけでは
  ありません。参考にしてください」 ⑨ライブ告知（毎晩21時） ⑩ハッシュタグ ⑪免責
- **サムネ**: 特大数字1つ（「+314万円」等）＋赤帯「答え合わせ」＋日付＋隅に「概算」。
  スマホで特大数字だけ読めれば勝ち
- **ツイート**: ✅箇条書き型（かおるさんのXの文体）。数字3-4個＋「参考程度に」の一文＋URL

## 7. 絶対ルール（判断の余地なし・すべて実際の失敗から）

1. **編集用素材にアイキャッチを焼き込まない**。かおるさんが再編集した動画から
   焼き込み文字は消せず、下の字幕も復元不能になった。「素材=クリーン」「完成版=焼き込み」を分離
2. **下部にオーバーレイを置かない**。本編の字幕・番組テロップと衝突する。日付は左上へ
3. **映像の日付と答え合わせの日付を混同しない**。下部/左上の日付は「映像がいつか」。
   株価の時点は注記で別に書く（前任は2025年の映像に2026年の日付を書き指摘された）
4. **お金の数字は全部python3で検算**し、表でユーザーに見せてから焼き込む
5. **現在の株価・配当・株数はユーザーのスクショを正**とする。WebSearchは暫定・要確認と明記
6. **チャンネル名・銘柄コードの取り違え禁止**（8306三菱UFJ/8316三井住友FG/8309三井住友トラスト
   は紛らわしい。四季報画面のコードで確認）
7. **納品後のお掃除**（deliverables削除）と**リリース素材の削除推奨**を忘れない
8. モデルIDや内部の識別子を動画・コミット・PRに書かない
