# 引き継ぎ書：バフェットかおる「答え合わせ切り抜き動画」の作り方

これはバフェットかおるさん（YouTube投資チャンネル運営・登録者27,500人）の切り抜き動画制作を
引き継ぐClaudeへの完全な指示書です。**まずこの文書を最後まで読んでから作業を始めてください。**

---

## 1. 目的 — 何のために作るのか

かおるさんのチャンネルの核メッセージはこれです：

> 高配当株はAIや半導体のように乱高下しない代わりに、上げ下げを繰り返しながら着実に右肩上がり。
> さらに「増配」で配当金は追加投資なしでも数年〜数十年で2倍・3倍、ときに10倍に育つ。
> いずれ配当金だけで投資元本を回収でき、老後は"タダで増えていく配当"を受け取るだけになる。

このメッセージを一番説得力ある形で伝えるのが**「答え合わせ動画」**です。
過去のライブ配信で「この銘柄に注目」「実際に買いました」と話した映像に、
**現在の株価・配当と突き合わせたアイキャッチを焼き込み**、
「かおるさんの投資が実際にうまくいっている」ことを視聴者が数字で実感できる動画にします。

- 過去の映像そのものが「予想時点の証拠」になる（画面に当時の株価・日付が映っていれば最強）
- 数字は実績なので盛る必要が一切ない。**盛らない・正直に**が信頼につながる
- 負けた銘柄も隠さず載せる（例：竹本容器+7%を灰色で表示）。全勝より「3勝1敗」の方が信頼される

## 2. 依頼主とチャンネルの背景

- 本人：元JAL客室乗務員。48歳から投資を始め5年で1億円、現在資産1.5億円超・年間配当500万円超
- メインチャンネル「バフェットかおる」（27,500人）：毎晩21時ごろライブ配信（経済ニュース＋個別株解説）
- 旧チャンネル「底辺CAセラの投資」（15,000人）：移行前のライブアーカイブが約270本ある宝の山
- 投資スタイル：日本の高配当株を長期保有（MS&AD・三井住友トラスト・ヒューリック・竹本容器・
  NTT・三菱商事・オカムラ・アネスト岩田・パラマウントベッドなど80銘柄超）
- ITは得意ではないが、スクリーンショットの提供とGitHubのドラッグ&ドロップ操作はできる。
  MacにローカルのClaude Codeもあり、YouTubeからのダウンロード作業はそちらに任せられる

## 3. 環境の制約と動画の受け渡し方法（重要）

クラウドのClaude Code環境は**YouTube・Googleドライブ・大半の外部サイトへの接続がブロック**されている。
使える経路は github.com / objects.githubusercontent.com と WebSearch のみ。

### 素材動画の受け取り（ユーザー→Claude）
1. ユーザーがYouTube Studioから動画をダウンロード（できない動画もある：著作権申し立て付き等。
   その場合はMacのローカルClaude Codeにyt-dlpでダウンロードさせる）
2. ユーザーが https://github.com/buffettkaoru/ending-note/releases/new にタグを作って
   ドラッグ&ドロップ→「Publish release」（下書きのままだと取得できないので必ずPublishまで案内）
3. Claudeが `curl -sSL --cacert /root/.ccr/ca-bundle.crt -o video.mp4 "https://github.com/buffettkaoru/ending-note/releases/download/<タグ>/<ファイル名>"` で取得
4. リリース一覧の確認: `curl -sSL "https://github.com/buffettkaoru/ending-note/releases/expanded_assets/<タグ>"` をgrep

### 完成動画の納品（Claude→ユーザー）
**SendUserFileのファイルカードはユーザー環境で開けないことがあった。GitHubリンク方式が確実：**
1. 完成mp4を `deliverables/` に**ASCIIファイル名**でコピーしてコミット＆プッシュ（100MB未満にすること）
2. `https://github.com/buffettkaoru/ending-note/raw/<ブランチ名>/deliverables/<ファイル名>` のリンクを送る
   （クリックだけで「ダウンロード」フォルダに保存される。curl -r 0-1000 で事前にリンク動作確認）
3. ユーザーの「できた」を確認したら `git rm` でお掃除コミット（リポジトリを汚さない約束）

### 監視
send_later（60分間隔）でreleasesページとPR状態を定期チェック。変化がなければユーザーに通知せず
静かに再スケジュール。アップロード待ちのときは45分に短縮してもよい。

## 4. 制作ワークフロー（10ステップ）

1. **受領・確認**：動画をダウンロード → `ffprobe` で長さ・解像度・音声を確認
2. **フレームスキャン**：1分ごと（長い動画は3分ごと）に1フレーム抽出し、タイムスタンプを
   焼き込んで4×5のタイル画像にまとめ、**Readツールで自分の目で見る**。
   目的：(a)全体の構成把握 (b)銘柄名・スライドの特定 (c)**当時の株価が映った画面（証拠画面）を探す**
   ```bash
   for m in $(seq 0 75); do ffmpeg -y -ss $((m*60+30)) -i video.mp4 -frames:v 1 \
     -vf "scale=400:225,drawtext=fontfile='$FONT':text='${m}min':fontsize=28:fontcolor=yellow:borderw=3:bordercolor=black:x=8:y=8" f$m.png; done
   # その後 concat + tile=4x5 で1枚に
   ```
3. **数字の収集**：
   - 当時の株価：**映像内の証券画面から読み取る**のが最優先（日付入りなら証拠として最強）
   - 現在の株価・配当：**必ずユーザーにSBI証券のスクショを依頼し、それを正とする**。
     WebSearchの株価は古い・混在していることがある（過去にヒューリックで1,875円と出たが
     実際は1,734.5円だった）。検索値を使う場合は「要確認」と明記する
   - 増配データ：IR BANK風の配当推移スクショをもらう（年度・金額・増配率が読める）
4. **検算（絶対にやる）**：騰落率・増配率・合計額をすべて自分で計算し直し、表にしてユーザーに提示。
   お金の数字なので1つの誤りが信頼を壊す。過去に合計額の暗算を間違えて指摘された。
   `python3 -c "print(3326/1983.1)"` のように必ず電卓を使う
5. **切り抜き区間の決定**：
   - 優先順位：①銘柄を一気に説明している区間（配分表・銘柄スライド） ②実際に購入する画面
     ③アナリティクスでチャットが盛り上がった時間帯（ユーザーにスクショをもらえる）
   - 通常動画は**8分超**にするとミッドロール広告が入れられる（収益目標があるので意識する）
   - 境界は `silencedetect`（noise=-30dB:d=0.4）で無音区間を探し、**文の切れ目にスナップ**
   ```bash
   ffmpeg -ss <候補-20> -t 40 -i video.mp4 -vn -af silencedetect=noise=-30dB:d=0.4 -f null -
   ```
6. **16:9無劣化カット**：`ffmpeg -ss <開始秒> -i video.mp4 -t <秒数> -c copy -movflags +faststart cut.mp4`
   （再エンコードしない。開始はキーフレームに吸着し数秒早まるが編集素材として問題ない）
7. **アイキャッチ焼き込み**（§5のデザイン仕様どおり）
8. **検証**：中間・冒頭・末尾のフレームを抽出してReadで目視（文字化け・重なり・誤字・数字を確認）。
   ffprobeで長さとサイズ（100MB未満）を確認
9. **納品**：§3の納品手順＋タイトル・概要欄・サムネ文言のセットを添える
10. **お掃除**：DL確認後にdeliverablesから削除。リリースの元動画削除も勧める（公開状態のため）

## 5. アイキャッチのデザイン仕様（意図つき・720p基準の実寸）

フォント：`FONT=$(fc-list :lang=ja --format '%{file}\n' | grep -i bold | head -1)`（Noto Sans CJK Bold。
なければ `apt-get install -y fonts-noto-cjk`）。
drawtextは**必ず textfile= + expansion=none** を使う（直書きだと「%」「&」「:」が化ける・消える）。
テキストは `printf '文言' > txt/tN.txt` で用意する。

| 位置 | 要素 | 色 | 意図 |
|---|---|---|---|
| 左上 | 動画の正体バッジ2行（例:「1年半前の銀行株予想」「本人が答え合わせ」） | 赤 0xCC0000@0.9・白文字 | 何の動画か・誰の動画かが2秒でわかる。「本人が」が重要（切り抜きチャンネルではなく本人という差別化） |
| 右上 | 見出し（例:「銀行株の答え合わせ」「ほったらかしで右肩上がり」） | 黄 yellow@0.92・黒文字 | 答え合わせであることの宣言 |
| 右上 | 銘柄ごとの結果「銘柄 当時→現在円 +○%」 | 勝ち＝緑 0x0B6E2F@0.88・白文字／負け・横ばい＝灰 0x666666 | 数字が主役。緑=成功が直感的。負けを灰色で正直に載せる |
| 右上 | 増配データ「配当 64→86→96円予」 | 金 0xFFC93C@0.9・黒文字 | 株価だけでなく「配当が育つ」ことを見せる（チャンネルの核メッセージ） |
| 下部中央 | 的中メッセージ（例:「金利上昇→利ざや改善の読みが的中」） | 金 0xFFC93C@0.92・黒文字 | 当時の論理が正しかったことを一言で |
| 下部中央 | **映像の配信日**「この映像は2025年2月17日のライブ配信」 | 赤茶 0xB33000@0.85・白文字 | **必須。** 視聴者が映像内の株価を現在と混同しないため。「1年前・1年半前の映像」だと伝わることが答え合わせの命 |
| 下部中央 | 注記「※株価・配当は2026年7月3日時点／予=会社予想」 | 黒 black@0.6・白文字・小さめ | 誠実さ。いつの数字での答え合わせかを明示 |

実寸（720p。他解像度は 高さ÷1080×係数 でスケール）：
左上1行目 fontsize36 x=27 y=27 / 2行目 27px y=89 / 右上見出し 30px y=27（右寄せ x=w-text_w-27）/
銘柄行 28px y=78,124…（46px間隔）/ 配当行 25px / 下部 金25px y=h-134・赤27px y=h-84・注記18px y=h-38。
boxborderw は フォントサイズの1/3 前後。

### 実際に使ったコマンド例（そのまま流用可）
```bash
D="fontfile='$FONT':expansion=none:fontcolor=white"
ffmpeg -y -i cut.mp4 -vf "\
drawtext=$D:textfile=txt/t1.txt:fontsize=36:box=1:boxcolor=0xCC0000@0.9:boxborderw=11:x=27:y=27,\
drawtext=$D:textfile=txt/t2.txt:fontsize=27:box=1:boxcolor=0xCC0000@0.9:boxborderw=8:x=27:y=89,\
drawtext=$D:textfile=txt/t3.txt:fontsize=30:fontcolor=black:box=1:boxcolor=yellow@0.92:boxborderw=8:x=w-text_w-27:y=27,\
drawtext=$D:textfile=txt/t4.txt:fontsize=28:box=1:boxcolor=0x0B6E2F@0.88:boxborderw=8:x=w-text_w-27:y=78,\
drawtext=$D:textfile=txt/t5.txt:fontsize=28:box=1:boxcolor=0x0B6E2F@0.88:boxborderw=8:x=w-text_w-27:y=124,\
drawtext=$D:textfile=txt/t6.txt:fontsize=25:fontcolor=black:box=1:boxcolor=0xFFC93C@0.9:boxborderw=7:x=w-text_w-27:y=170,\
drawtext=$D:textfile=txt/t7.txt:fontsize=25:fontcolor=black:box=1:boxcolor=0xFFC93C@0.9:boxborderw=7:x=w-text_w-27:y=212,\
drawtext=$D:textfile=txt/t10.txt:fontsize=25:fontcolor=black:box=1:boxcolor=0xFFC93C@0.92:boxborderw=9:x=(w-text_w)/2:y=h-134,\
drawtext=$D:textfile=txt/t8.txt:fontsize=27:box=1:boxcolor=0xB33000@0.85:boxborderw=10:x=(w-text_w)/2:y=h-84,\
drawtext=$D:textfile=txt/t9.txt:fontsize=18:box=1:boxcolor=black@0.6:boxborderw=6:x=(w-text_w)/2:y=h-38" \
-c:v libx264 -preset veryfast -crf 20 -pix_fmt yuv420p -c:a copy -movflags +faststart final.mp4
```
縦型ショート（1080×1920・上下ぼかし）が必要な場合は同フォルダの `extract_short.sh` を使う。

## 6. 日付と数字のルール（過去の失敗から学んだこと）

1. **下部の日付は「映像がいつのものか」を書く。** 編集した日や新しい配信日を書かない。
   （失敗例：2025年のライブ映像に「2026年7月4日の配信より」と書いて指摘された。
   「1年前の映像だ」と伝わることが答え合わせ動画の生命線）
2. 答え合わせに使った株価の日付は注記に別途書く（「※騰落率は2026年7月3日時点の株価で計算」）
3. すべての数字を検算してから焼き込む。ユーザーに検算結果の表を見せて確認をとるのが理想
4. 配当の前提をユーザーが指定したら（例:「4.4%が2回・2回目は10%増配」）、そのとおり計算しつつ
   概要欄に前提の注記を必ず入れる
5. WebSearchで得た株価・情報は暫定。ユーザーのスクショと矛盾したら**スクショが正**

## 7. タイトル・概要欄・サムネのテンプレ（答え合わせ型）

- **タイトル**：【答え合わせ】＋実額・実%＋期間。例：
  「【答え合わせ】1年前に100万円で爆買いした高配当株4銘柄→配当もらいながら143万円になっていました」
  「【答え合わせ】バフェットが5,451億円調達した頃に注目した銀行株→三菱UFJ+68%・三井住友FG+72%【2025.2.17ライブ】」
- **概要欄の構成**：①いつ何を話した映像か → ②銘柄ごとの当時→現在（箇条書き）→ ③増配データ →
  ④核メッセージ（乱高下しない/増配で育つ/いずれ元本回収）→ ⑤チャプター → ⑥ライブ告知（毎晩21時）→
  ⑦ハッシュタグ（#高配当株 #答え合わせ #新NISA #配当金生活 #増配 等）→ ⑧免責（時点・自己責任・推奨ではない）
- **サムネ文字**：特大の数字1つ（「+72%」「100万円→143万円」）＋赤帯「答え合わせ」＋日付「2025.2.17→2026.7.3」。
  スマホで特大数字だけ読めれば勝ち。3ブロックまで
- 投稿は毎晩21時のライブ前（19〜20時台）がライブへの導線になる

## 9. ユーザーとのやりとりのコツ

- 専門用語を避け、番号つきの短い手順とリンクで案内する（「Choose a tagをクリック→live-videoと入力」レベルまで具体的に）
- 質問より提案。ただし**お金の数字と日付だけは必ず確認**をとる
- ユーザーはスクショでの情報提供が得意。株価・配当・アナリティクスはスクショを依頼する
- 詰まったら「その画面のスクショを送ってください」で解決する
- 納品後に「できた」をもらってからリポジトリをお掃除。約束したお掃除は必ず実行する
- 作業中も自動チェック（send_later）を絶やさず、変化がないときは静かに待つ
