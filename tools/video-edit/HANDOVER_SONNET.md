# 動画編集 引き継ぎ書【Claude Sonnet 用】

作成: Claude Fable 5(2026-07-06)/ 実績: 2025/12/23収録「株式分割」回の編集(PR #8)

この文書は**手順書**である。順番どおりに実行し、各STEPの✅ゲートを通過するまで次に進まないこと。
判断に迷ったら、創作せずユーザーに選択肢を示して聞く。**特に⚠マークは実際に事故った箇所**。

## STEP 0: 環境準備

```bash
apt-get update && apt-get install -y ffmpeg fonts-noto-cjk
```
⚠ `apt-get update` を省くと404で失敗する。

日本語フォントのパスを確認(どちらかが存在すればOK):
```bash
ls /usr/share/fonts/truetype/fonts-japanese-gothic.ttf \
   /usr/share/fonts/opentype/noto/NotoSansCJK-Bold.ttc 2>/dev/null
```

**知っておくこと:**
- ここはリモートコンテナ。ユーザーのPCのファイル(ダウンロードフォルダ等)には**アクセスできない**。
- YouTubeはプロキシでブロックされている。yt-dlpは**動かない**。試行3回で打ち切り、粘らない。
- 作業ファイルはすべてスクラッチパッドディレクトリに置く。リポジトリを汚さない。

✅ゲート: `ffmpeg -version` が表示され、フォントファイルが1つ以上存在する。

## STEP 1: 動画の入手(GitHubリリース経由のみ)

1. ユーザーに依頼:「リポジトリの Releases → `live-video` タグ → 動画ファイルを
   ドラッグ&ドロップ → Publish」(1ファイル2GBまで)
2. GitHub MCPの `get_release_by_tag` でアセット名とsha256 digestを取得。
3. ダウンロードと検証:
```bash
curl -L --cacert /root/.ccr/ca-bundle.crt -o INPUT.mp4 \
  "https://github.com/<owner>/<repo>/releases/download/<tag>/<file>.mp4"
sha256sum INPUT.mp4   # ← リリースAPIのdigestと一致必須
ffprobe -v error -select_streams v:0 -show_entries stream=width,height -of csv=p=0 INPUT.mp4
ffprobe -v error -show_entries format=duration -of csv=p=0 INPUT.mp4
```
⚠ ffprobeの出力区切りに `s=' '` を指定するとクォートが壊れる。`csv=p=0`(カンマ区切り)を使い、
bashでは `IFS=',' read -r W H` で受ける。

✅ゲート: sha256一致、解像度・長さが取得できている(例: 1280x720, 713.5秒)。

## STEP 2: 内容分析(省略禁止。ここをサボると誤った編集をする)

### 2-1. 全体俯瞰(30秒ごとのフレームをグリッド1枚に)
```bash
mkdir -p frames
for t in 5 30 60 90 120 150 180 210 240 270 300 330 360 390 420 450 480 510 540 570 600 630 660 690; do
  ffmpeg -y -v error -ss $t -i INPUT.mp4 -frames:v 1 -q:v 3 frames/f_$(printf %03d $t).jpg
done
ffmpeg -y -v error -pattern_type glob -i "frames/f_*.jpg" \
  -filter_complex "scale=512:288,tile=5x5:margin=2:padding=2" allgrid.jpg
```
→ allgrid.jpg を**Readで見る**。各フレームから読み取ってメモすること:
- [ ] 銘柄名(スライドの文字から全部書き出す)
- [ ] トピックの一覧と順序
- [ ] 焼き込み字幕の有無と位置
- [ ] 本編がどこで終わるか / 終了後に何があるか

⚠ 後半に見た目の違う映像(風景・音楽など)があっても**「不要部分」と決めつけない**。
今回は後半4分が「JALの歌」というオリジナル曲コーナー(意図的な内容)だった。判定方法:
```bash
# 話し声区間は0.5秒以上の無音が頻出、歌・BGM区間はゼロになる
ffmpeg -ss 480 -t 180 -i INPUT.mp4 -af "silencedetect=noise=-30dB:d=0.5" -f null - 2>&1 | grep -c silence_start
```

### 2-2. トピック境界の特定
```bash
ffmpeg -i INPUT.mp4 -vf "select='gt(scene,0.08)',showinfo" -f null - 2>&1 | grep pts_time
```
フェード切替は漏れるので、怪しい区間は追加フレーム抽出で±2秒まで詰める。

### 2-3. 字幕位置の実測(日付バッジの位置決めに必須)
**長い字幕が出ているフレームを原寸でReadし、字幕の上端yと左端xを読み取る。**
⚠ 字幕は中央寄せでも、長文は画面左端近く(720pでx≈85)まで届く。
「左下の隅なら被らない」は**誤り**。バッジは字幕帯の「すぐ上」に置く。
(2025/12/23回の実測: 字幕帯は y=615〜700 → バッジは y=572)

✅ゲート: 銘柄リスト・トピック境界表(開始秒つき)・字幕上端yの3点が揃っている。

## STEP 3: オーバーレイ設計

### 3-1. 日付バッジ(左下・常時)
| 項目 | 値(720p) | 一般式 |
|---|---|---|
| テキスト | `2025/12/23 収録` | 収録日は**ユーザー指定の日付**を使う。創作禁止 |
| フォントサイズ | 26 | H/28 |
| 色 | 白文字 / `black@0.55` 箱 / `boxborderw=10` | 固定 |
| 位置 | x=16, y=572 | yは「字幕上端 − バッジ高さ − 数px」を**実測から計算** |

### 3-2. アイキャッチ(右上・トピック冒頭のみ)
| 項目 | 値(720p) | 備考 |
|---|---|---|
| フォントサイズ | 30 | H/24 |
| 位置 | `x=w-tw-28:y=28` | 右上固定 |
| 表示時間 | 12〜15秒 | トピック冒頭に置く。常時表示はしない |
| フェード | 0.4秒 | 下のフィルタ雛形に含む |
| 密度 | 約1個/分 | 11分なら11〜14個 |
| 文言 | 12文字以内・`★ `プレフィックス | 銘柄名を優先的に入れる |

**色の使い分け(boxcolor):**
- 赤 `0xD32F2F@0.85` = 注意・重要・テーマ(デフォルト。迷ったらこれ)
- 緑 `0x2E7D32@0.85` = 上昇・増加(「分割後に上昇した銘柄」「配当が増えた」)
- 青 `0x1565C0@0.85` = 基礎知識・戦略の解説
- 金 `0xF9A825@0.90` + 黒文字 = 配当金・資産額の金額ハイライト

**「投資額・配当金が増えていく」演出**(該当する内容がある場合のみ):
```
"5:00-5:12|配当金 年15,000円"     → 緑
"5:12-5:24|配当金 年31,000円 ↑"   → 緑
"5:24-5:40|配当金 年52,000円 ↑↑"  → 金(最終形を強調)
```
- 数字は**動画内で実際に言及された値のみ**。自分で計算・推定した数字は使わない。
- 成長感を大きさで出す場合は fontsize 30→34→38 の3段まで(それ以上は禁止)。
- 「↑」等の記号は STEP 4-0 の単体テストで豆腐化しないことを確認してから使う。

### 3-3. ラベル文言のルール
- ⚠ **半角コロン `:` を絶対に入れない**(理由はSTEP 4-0)。区切りは「」・全角スペース・「・」。
- 銘柄名は正式短縮形(伊藤忠商事/ブリヂストン/ショーボンド/NTT/トヨタ/ソニーG/任天堂/メルカリ/ZOZO)。
- 内容を確認できないラベルを創作しない。スライド・字幕に書いてある言葉から作る。

✅ゲート: 「開始-終了|ラベル|色」の一覧表が完成し、境界±2秒の根拠(フレーム)がある。

## STEP 4: フィルタ構築とテスト

### 4-0. ⚠ 最重要の既知バグ2件(必ず先に読む)

**バグ1: drawtextのtext内の半角コロン**
text値に `:`(ASCII 0x3a)があると、クォートしていてもパースが壊れ、
テキストが途中で切れた上に**日本語が全部豆腐(□)になる**。
- 対策: ラベルに `:` を使わない。どうしても必要なら `\:` にエスケープ。
- ⚠ 全角コロンを入れたつもりがASCIIになっている事故が実際に起きた。疑ったら:
  `echo "ラベル" | od -t x1z` → 全角 `:` は `ef bc 9a`、ASCII `:` は `3a`。

**バグ2: -ss の位置でタイムスタンプが変わる**
`-ss` を `-i` の**前**に置くとt=0リセットされ、`enable='between(t,S,E)'` が発火しない。
- 時刻依存フィルタの確認は必ず: `ffmpeg -i INPUT.mp4 -ss <秒> -vf "$FILTER" -frames:v 1 out.jpg`
  (`-i` の**後**に `-ss`。遅いが正確)

### 4-1. フィルタ雛形
```
# 日付バッジ(FONT・y は実測値に置換)
drawtext=fontfile='FONT':text='2025/12/23 収録':fontcolor=white:fontsize=26:box=1:boxcolor=black@0.55:boxborderw=10:x=16:y=572

# アイキャッチ1個分(S,E=秒、LABEL=★付き文言、COLOR=boxcolor)
,drawtext=fontfile='FONT':text='LABEL':fontcolor=white:fontsize=30:box=1:boxcolor=COLOR:boxborderw=14:x=w-tw-28:y=28:enable='between(t\,S\,E)':alpha='if(lt(t\,S+0.4)\,(t-S)/0.4\,if(gt(t\,E-0.4)\,(E-t)/0.4\,1))'
```
実装例は `tools/video-edit/edit_video.sh` の filter構築部を流用してよい。

### 4-2. 文字化けテスト(本番前に必ず)
```bash
ffmpeg -y -v error -f lavfi -i color=gray:size=800x200:duration=0.1 \
  -vf "drawtext=fontfile='FONT':text='★ 伊藤忠商事 3つの盤石 ↑':fontcolor=white:fontsize=30:x=20:y=80" \
  -frames:v 1 glyph_test.png
```
→ Readで見て、**全文字が正しく描画されている**こと(豆腐・欠けが1文字でもあれば原因除去)。

### 4-3. 配置テスト(4フレーム)
「長い字幕の場面」「大きいスライドの場面」「アイキャッチ表示中」「終盤」の4時刻で
`-i INPUT.mp4 -ss T` 方式でフレームを出し、グリッド合成してReadで目視。

✅ゲート: 4フレームすべてで (a)バッジが字幕・スライド本文に非接触 (b)アイキャッチが正しい文言で表示。
1つでもNGなら設計に戻る。**NGのまま本番レンダしない。**

## STEP 5: 本番レンダリング

```bash
ffmpeg -y -i INPUT.mp4 -vf "$FILTERS" \
  -c:v libx264 -preset medium -crf 20 -c:a copy -movflags +faststart OUTPUT.mp4
```
- 音声は必ず `-c:a copy`(無劣化)。
- 目標サイズ100MB未満(GitHub pushの上限)。720p/12分でcrf20≒89MBの実績。
  超えそうなら: 総ビットレート = 95MB×8÷秒数、そこから音声分を引いて `-b:v` 2パス。

✅ゲート: `ffprobe` で duration が元動画と一致(±0.1秒)、サイズ<100MB。

## STEP 6: 全数検証(スキップ禁止)

出力ファイルから **全アイキャッチの中間時刻 + アイキャッチが無いはずの時刻2点** を抽出:
```bash
for t in <中間時刻リスト> <非表示時刻2点>; do
  ffmpeg -y -v error -ss $t -i OUTPUT.mp4 -frames:v 1 -q:v 4 frames/v_$t.jpg
done
# tile=4x4等でグリッド合成してReadで目視
```
チェック項目(全フレーム):
- [ ] アイキャッチの文言が設計表と完全一致(豆腐・切れなし)
- [ ] 日付バッジが全フレームに出ている
- [ ] 字幕・スライド本文に被っていない
- [ ] 非表示時刻のフレームにアイキャッチが残っていない

✅ゲート: 全項目OK。1つでもNGなら修正して**STEP 5からやり直し**(部分修正で済ませない)。

## STEP 7: 納品

1. `SendUserFile`(display:attach)で動画本体、(display:render)で検証グリッド画像を送る。
2. リポジトリの作業ブランチに `video/<名前>_edited.mp4` としてcommit & push(<100MBのみ)し、
   `https://github.com/<owner>/<repo>/raw/<branch>/video/<file>` のリンクを渡す。
   コミットメッセージに「マージ前に削除予定」と明記。
3. 報告文には**タイムスタンプ×ラベルの一覧表**を必ず含める(ユーザーが修正指示を出しやすい)。
4. ユーザーがダウンロード完了と言ったら、動画バイナリをブランチから削除する。
5. タイトル・概要欄を頼まれたら、概要欄1行目は必ず日付注釈:
   `※この動画は2025年12月23日に収録したものです。株価・分割情報は収録時点のものです。`
   目次はアイキャッチのタイムスタンプから生成(`0:00` 開始必須)。

## STEP 8: 切り抜きを頼まれた場合

1. **1クリップ=1テーマ**。トピック境界表から「単独で成立する山」を選ぶ
   (例: 「分割後も低迷した銘柄」1:55-2:27、「伊藤忠3つの盤石」3:28-4:26)。
2. IN/OUT点は**無音位置**から選ぶ:
   `ffmpeg -i INPUT.mp4 -af "silencedetect=noise=-30dB:d=0.3" -f null -` の無音リストから、
   テーマ開始直前・終了直後の無音を採用。⚠ 文の途中で切らない。
3. 冒頭3秒に結論系アイキャッチを焼き込む(そのクリップの答えを先に見せる)。
4. 長さ: 通常45〜90秒 / Shorts(9:16)は60秒以内。
5. Shorts化は crop でなくレイアウト再構成: 上1/3=スライド部分のcrop、下2/3=人物部分のcrop。
   スライド・人物の座標は毎回フレームで実測(動画によって違う)。
6. **切り抜きにも日付バッジ必須**(文脈が切れる分、収録日の明示がより重要)。
7. カット編集は `-c copy` では正確に切れない(キーフレーム単位になる)。再エンコード
   (crf 20)で正確に切る。
8. 末尾5秒に「フル版は概要欄へ」等の誘導テキストを入れる。

## STEP 9: してはいけないこと

- ユーザーのPCのファイルを探し続ける/YouTubeダウンロードを繰り返し試す
- 動画にない銘柄・数字・主張をラベルに書く(投資情報の創作は信頼を破壊する)
- 検証ゲートを飛ばして本番レンダ・納品する
- 後半の"見た目が違う映像"を確認なしにカットする
- 音声を再エンコードする(`-c:a copy` 以外を使う)
- 100MB超のファイルをgit pushする/mainに直接pushする
- 収録日を推測で書く(必ずユーザー指定の日付)
