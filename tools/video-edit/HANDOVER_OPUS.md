# 動画編集 引き継ぎ書【Claude Opus 用】

作成: Claude Fable 5(2026-07-06)/ 実績: 2025/12/23収録「株式分割」回の編集(PR #8)

あなた(Opus)は高い判断力を持つ前提で、この文書は「原則」と「品質基準」を中心に書く。
細部の判断は任せるが、**§5の落とし穴と§6の検証ゲートだけは絶対に省略しないこと**。

## 1. ミッションと成功の定義

バフェットかおるさん(投資系YouTuber)の動画に、視聴者の理解を助けるオーバーレイを入れる。

成功とは:
- **収録日が常に見える**(左下バッジ)。投資動画では「いつ時点の情報か」が信頼性の生命線。
  株価・分割情報・配当は日々変わる。日付のない投資動画は視聴者を誤解させる。
- **「今どの銘柄・どの話題か」が一目で分かる**(右上アイキャッチ)。
  視聴者は途中から見る・ながら見する。銘柄名(伊藤忠商事、NTT、メルカリ…)が
  切り替わった瞬間にアイキャッチで示すと、内容の流れがスムーズに伝わる。
- **元動画の焼き込み字幕・スライドを一切潰さない**。
- **音声無劣化・画質ほぼ維持・100MB以下**(GitHub経由で納品できるサイズ)。

## 2. 実行環境の現実(最初に必ず確認)

- リモートコンテナである。**ユーザーのPC(ダウンロードフォルダ)には絶対に届かない**。
  「動画はダウンロードにあります」と言われたら、それはユーザーのローカルの話。
- **YouTubeはプロキシで403ブロック**。yt-dlpは使えない。粘らない。
- 動画の入手ルートは **GitHubリリースアセット**(2GBまで)。ユーザーに
  `リポジトリ → Releases → 既存の live-video タグ → ファイルをドラッグ&ドロップ` を案内する。
  `github.com/<owner>/<repo>/releases/download/...` へのcurlは通る。sha256はリリースAPIの
  digestと照合して破損を検出する。
- ffmpegは未インストール。`apt-get update && apt-get install -y ffmpeg fonts-noto-cjk` を最初に実行
  (updateを忘れると404で失敗する)。日本語フォントは
  `/usr/share/fonts/truetype/fonts-japanese-gothic.ttf`(IPA)か Noto CJK。
- 納品は SendUserFile(チャット添付)+ GitHubブランチへのpush(rawリンク提示)の二段構え。
  ユーザーはファイルカードに気づかないことがある。「どこにあるの?」と聞かれたら
  両方のルートを丁寧に案内する。

## 3. 編集の9割は「内容理解」(ここで手を抜くと全てが台無し)

オーバーレイを入れる前に、動画を"見る"こと。あなたは音声を聞けないが、以下で内容を掴める:

1. `ffprobe` で解像度・長さ・ビットレートを取る。
2. **30秒ごとにフレーム抽出 → `scale=512:288,tile=5x5` で1枚のグリッドに合成 → 画像として読む。**
   これで11分の動画が1〜2枚の画像で俯瞰できる。スライドの文字・焼き込み字幕から
   銘柄名とトピックがほぼ全部読み取れる。
3. スライド切替の正確な時刻は scene検出(`select='gt(scene,0.08)'`)。フェード切替は
   検出漏れするので、怪しい区間は追加フレームで挟み撃ちして境界を±2秒まで詰める。
4. **音声の性質は silencedetect で推定**: 講義区間は0.5秒以上の無音(話の間)が頻出、
   歌・BGM区間はゼロ。今回これで「後半4分の飛行機映像はJALのオリジナル曲コーナー
   (意図的なコンテンツ)」と判定し、誤ってカットせずに済んだ。
   **見た目が本編と違う=不要部分、と決めつけないこと。**

## 4. デザイン仕様(720p実測値。他解像度は比例)

### 左下: 収録日バッジ(常時表示)
- テキスト `2025/12/23 収録`、白文字、fontsize = H/28(720pで26)、`box=1:boxcolor=black@0.55:boxborderw=10`
- 位置 x=16。**yは必ず実フレームで字幕の上端を実測してから決める**(720pで572だった)。
  焼き込み字幕は中央寄せでも長文はほぼ全幅(x≈85まで)に達する。「左下の隅なら字幕と
  被らない」は誤り。字幕帯の"すぐ上"に置くのが正解。
- 黒@0.55はスライドの白地でも人物背景でも読める万能色。日付は主役ではないので控えめに。

### 右上: アイキャッチ(トピック冒頭に12〜15秒、フェード0.4秒)
- `★ `プレフィックス + 白文字、fontsize = H/24(720pで30)、`boxborderw=14`、x=w-tw-28、y=28
- 表示密度はおおむね**1分に1個**。トピックの数だけ入れる(今回13個)。常時表示はしない
  ——出た瞬間に目を引くから「アイキャッチ」である。
- フェードは `alpha='if(lt(t,S+0.4),(t-S)/0.4,if(gt(t,E-0.4),(E-t)/0.4,1))'` + `enable='between(t,S,E)'`

### 色の意味(投資動画の文法)
- **赤 0xD32F2F@0.85**: 注意・重要・テーマ提示(デフォルト)
- **緑 0x2E7D32@0.85**: 上昇・増加・成長(「分割後に上昇した銘柄」「配当が増えた」)
- **青 0x1565C0@0.85**: 冷静な解説・基礎知識・戦略
- **金 0xF9A825@0.90**(文字は黒推奨): お金・配当金・資産額のハイライト
- 今回は全部赤で統一したが、次回からは意味で塗り分けると更に良い。

### 「増えていく」を伝えるアイキャッチ(配当・資産の成長演出)
数字の変化はテキストに含めて時系列で連打する。例:
```
"5:00-5:12|配当金 年15,000円"       (緑)
"5:12-5:24|配当金 年31,000円 ↑"     (緑・↑付き)
"5:24-5:40|配当金 年52,000円 ↑↑"    (金・最終形を強調)
```
- 数字は動画内でかおるさんが実際に言っている値だけを使う。**勝手に計算・創作しない。**
- 大きさで成長を表すなら fontsize を段階的に上げる(30→34→38)。ただし1.3倍まで。
  それ以上は下品になり、スライドに被る。
- 「↑」「→」はフォントにより豆腐化しうる。**必ず事前に単体レンダで確認**(§5参照)。

### ラベル文言の作法
- 12文字以内が目安(長いとスライドの上に被る)。銘柄名は正式短縮形
  (伊藤忠商事、ブリヂストン、ショーボンド、NTT、トヨタ)。
- 区切りは「」・全角スペース・「・」を使う。**半角コロンは絶対に使わない**(§5)。

## 5. 実際に踏んだ落とし穴(全部再発しうる。必ず読む)

1. **drawtextのtext内の半角コロン`:`はオプション区切りと衝突**し、テキストが切り詰められた上に
   フォント指定が壊れて日本語が全部豆腐になる。クォートでも防げなかった。
   対策: ラベルに`:`を入れない(全角`:`か「」を使う)。エスケープするなら`\:`。
   さらに罠: エディタ上で全角に見えても実体がASCIIのことがある。**`od -t x1z`でバイト確認**
   (全角コロンは `ef bc 9a`、ASCIIは `3a`)。
2. **`-ss`を`-i`の前に置くとタイムスタンプが0リセット**され、`enable='between(t,...)'`が発火しない。
   時刻依存フィルタのテストは必ず `-i INPUT -ss T`(出力側シーク)で行う。遅いが正確。
3. ffprobeの `csv=p=0:s=' '` はシェルクォートが剥がれて壊れる。`csv=p=0` + `IFS=',' read` が安全。
4. フォント不安(★・↑・特殊記号)は、colorソース1枚に drawtext して目視確認してから本番へ。
5. apt install ffmpeg は `apt-get update` なしだと404で失敗する。

## 6. 検証ゲート(スキップ禁止)

1. **設計検証**: 本番前に代表4フレーム(字幕が長い場面・スライドが大きい場面を含める)へ
   フィルタを当て、グリッド合成して目視。日付バッジが字幕・スライド本文に被っていないか。
2. **全数検証**: 本番レンダ後、**全アイキャッチの中間時刻 + アイキャッチが無いはずの時刻2点**の
   フレームを出力ファイルから抽出しグリッドで目視。文字化け1つでも全滅扱いでやり直す。
3. duration が元と一致すること、音声が `copy` であることを ffprobe で確認。
4. 検証グリッド画像はユーザーにも送る(仕上がりが一目で伝わり、信頼につながる)。

## 7. 納品と後始末

- 出力: `libx264 -preset medium -crf 20 -c:a copy -movflags +faststart`。720p/12分でcrf20 ≈ 89MB。
  100MBを超えそうなら2パスでビットレート逆算(95MB×8/秒数 − 音声128k)。
- SendUserFile + ブランチに `video/` でpush(100MB未満のみ)し、rawのダウンロードURLを渡す。
- **マージ前にブランチから動画バイナリを削除する**(履歴汚染防止)。ユーザーのダウンロード
  完了を確認してから。
- タイトル・概要欄・サムネ文字を頼まれたら: 概要欄の1行目に必ず
  `※この動画は2025年12月23日に収録したものです。株価・分割情報は収録時点のものです。`
  の形式で日付注釈。目次(チャプター)はアイキャッチのタイムスタンプから作れる。0:00開始必須。

## 8. 切り抜き(クリップ)のコツと考え方

切り抜きを頼まれたら、以下の思考順で:

1. **切り抜きの目的は「1テーマ1動画」**。フル動画の要約ではない。今回の動画なら
   「メルカリ・ZOZOは分割後も低迷した」(1:55〜2:27)や「伊藤忠3つの盤石」(3:28〜4:26)が
   それぞれ独立した切り抜きになる。
2. **切り出し点は無音で切る**。silencedetect(-30dB, 0.3秒)で話の切れ目を列挙し、
   その中からIN/OUTを選ぶ。文の途中で切るのは最悪。
3. **冒頭3秒がフック**。結論・驚き・銘柄名が最初に来る位置から始める。導入の「えー、それでは」
   から始めない。必要なら冒頭にアイキャッチ(そのクリップの結論)を3秒焼き込む。
4. 長さの目安: 通常切り抜き45〜90秒、Shorts(9:16)は60秒以内。
5. **9:16化**は crop ではなくレイアウト再構成が基本: 上1/3にスライド(crop)、下2/3に人物(crop)、
   字幕は焼き込み済みなので人物cropに含める。スライドと人物の位置は動画ごとに実測する。
6. 切り抜きにも**必ず収録日バッジを入れる**。切り抜きは文脈が飛ぶ分、日付の重要性が増す。
7. 元動画への誘導テキスト(「フル版は概要欄へ」)を末尾5秒に入れると親切。

## 9. 仕事の進め方(ユーザーとの関係)

- ユーザーは非エンジニア。専門用語を使わず、**画像で見せる**(グリッド、レイアウト確認図)。
- 中間成果を早く見せる。「レイアウトはこうなります」の1枚が、100行の説明に勝る。
- タイムスタンプとラベルの一覧表を必ず示し、修正を1往復で受けられる形にする。
- 事実に忠実に。銘柄・数字は動画にあるものだけ。投資助言はしない(表示の整理に徹する)。
- PR運用: ブランチ `claude/video-eye-catch-edit-*` で作業、ドラフトPR、send_laterで1時間ごとの
  自己チェックを再武装。変化がなければ静かに終了。
